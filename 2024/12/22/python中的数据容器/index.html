在进行ssti注入的时候，我们的payload中会涉及到python中相关的数据容器，熟悉掌握这些数据容器，才能避免我们在语法上面出现错误和借助数据容器的一些特性进行绕过。

python中的数据容器就是可以一次性储存多个数据的一种数据结构，他和一般语言中的数组不一样，他可以储存不同类型的数据在同一个容器当中。而且他支持嵌套储存，也就是他的储存的元素本身也可以是一个数据容器。而且python中的数据容器有很多种类型，支持不同场景下面的应用，还有绝大多数的数据容器储存大小是动态的。python中所有的数据容器本质上都是通过一个类定义的，所以我们在定义对应的数据容器的时候都可以通过创建一个实例的方法来定义。

列表（list)
基础知识

列表的定义：

lis = [[1,2],2,3]

列表是直接使用[]进行定义。

列表中对应元素的访问：

lis[1] #2，lis[-1] #3，lis[0][0] #1

我们对列表中的元素进行访问的时候，可以从左向右进行访问，这个时候索引就是从0开始，我们也可以选择从右向左访问，这个时候索引就是从-1开始；如果是嵌套进行储存的话，我们直接递归进行访问就可以了。

我们还可以使用切片的方式来获取列表中的元素。

列表的特点：

列表中储存的数据上限是2^63-1，可以容纳不同类型的数据，是有序存储的（有下标序号在），允许重复数据存在，可以修改。

列表中常用的方法
元组（tuple)
基础知识

元组的的定义：

tup = (1,2,3)

元组是直接使用()进行定义的。

tup = ("hello")，tup = ("hello",)

在这里前面一个的数据类型是字符串，后面一个才是元组。所以在定义元组的时候，如果只定义了一个元素，我们需要在后面加上,。

元组中定义元素的访问：

和列表一样。

元组的特点：

和列表相比，就是元组中的元素不可以进行修改。但是如果元组中嵌套定义了其他数据容器，如列表，那么这个列表中的元素是可以进行修改的。

元组中常用的方法
字符串（str)
基础知识

字符串可以看成是储存字符的数据容器。

字符串的定义：

str = “you and me”

直接使用引号定义定义即可

字符串中的字符索引：

和列表一样

字符串的特点：

和列表相比，字符串的长度任意（取决于内存大小），只能存储字符，不可以进行修改。

集合（set)
基础知识

s = {1,2,3}

集合是直接使用{}进行定义的。

获取集合中的元素：

不能使用下标索引获取集合中的元素。

遍历获取或者使用pop方法进行获取。或者将集合转换为元组或者列表再进行获取。

集合的特点：

和列表相比，集合是无序的，所以集合不能使用下标去进行索引，还有集合中不能有重复元素；和元组相比，集合还有一特点就是可以进行修改。

集合中常用的方法
字典（dict）
基础知识

字典的定义：

dic = {key1:value1,key2:value2}

字典也是使用{}进行定义的，只不过字典中的元素是以键值对的方式储存。

字典中元素的定义：

不能使用下标索引获取集合中元素。

我们是通过键来获取对应的值。

字典的特点：

字典中的键和值可以是任意数据类型（键不能是字典），字典中没有重复键名（如果重复定义，后面值的会覆盖掉前面的值），字典是通过键值对的方式来进行访问，而不是下标索引，字典是可以进行修改的。

总结

  • 是否支持下标索引

    • 支持：列表，元组，字符串 — 序列数据
    • 不支持：集合，字典 — 非序列数据

  • 是否支持重复元素：

    • 支持：列表，元组，字符串 — 序列数据
    • 不支持：集合，字典 — 非序列数据

  • 是否可以修改：

    • 支持：列表，集合，字典
    • 不支持：元组，字符串

  • 是否有序：

    • 是：列表，元组，字符串
    • 不是：集合，字典

  • 支持数据类型：

    字符串只支持字符，字典中的键和值可以是任意数据类型（键不能是字典），其他的支持任意数据类型。
